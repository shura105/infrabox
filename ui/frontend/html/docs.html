Огляд системи

Infrabox — домашня SCADA-система для моніторингу та управління інженерними підсистемами об'єкта. Дані збираються через MQTT, зберігаються в Redis і відображаються на налаштовуваних екранах у реальному часі через WebSocket.

Основні можливості

  • Відображення значень параметрів у реальному часі
  • Налаштовувані SCADA-екрани з довільним розташуванням елементів
  • Ієрархія екранів із навігацією між ними
  • Кольорова індикація якості сигналу
  • Архівування всіх параметрів із побудовою трендів
  • Симулятор для тестування без реального обладнання

Архітектура

Система розгорнута у Docker-контейнерах та складається з кількох незалежних сервісів.

🌐 infrabox-web (nginx)
:80 → :443 · HTTPS-точка входу

Роздає статику UI, проксіює API, WebSocket та архіватор.

⚙ infrabox-backend
:8090 · FastAPI + uvicorn

REST API для екранів та проекту. WebSocket-стрім даних. Керування симулятором.

↕
🗄 infrabox-redis
:6379

Зберігає поточні значення всіх точок (point:NNN). Pub/Sub для push-оновлень.

🦟 infrabox-mosquitto-real
:1883 · MQTT broker

Приймає дані від реального обладнання.

🤖 infrabox-simulator
mosquitto-sim :1884

Генерує тестові дані замість реального обладнання.

↕
📦 infrabox-arch
архіватор

Записує всі зміни параметрів у файлові томи. UI доступний через /arch/.

🔑 infrabox-auth
:8095

Авторизація. JWT-токени зберігаються в cookie.

Маршрути nginx

  • / — головний екран перегляду
  • /constructor — конструктор екранів
  • /arch/ — архіватор
  • /api/ — backend API
  • /ws — WebSocket стрім даних
  • /auth/ — авторизація

Перегляд екранів

Головна сторінка (/) відображає SCADA-екрани. При завантаженні автоматично відкривається кореневий екран root.

Навігація

  • Кнопки nav_btn та картки Підсистема переходять на інший екран
  • Кнопка ← в хедері повертає на попередній екран
  • Назва поточного екрану відображається по центру хедера

Індикатор зв'язку

Правий нижній кут показує статус WebSocket-з'єднання:

  • ● live — дані надходять у реальному часі
  • ● error — з'єднання перервано, виконується повторне підключення
  • ● connecting — встановлюється з'єднання

Меню

Кнопка ☰ Меню у правій частині хедера відкриває навігаційне меню з доступом до конструктора, адміністрування, налаштувань та документації.

Конструктор екранів

Сторінка /constructor дозволяє створювати та редагувати SCADA-екрани. Доступна тільки авторизованим адміністраторам.

Структура проекту

Ліва панель показує дерево екранів. Кожен екран має унікальний ID (латинські літери, цифри, -, _) та може мати батьківський екран для логічної ієрархії. Файли зберігаються плоско: screens/{id}/screen.json.

Редагування елементів

  • Додати — вибрати тип у палітрі, клікнути на полотно
  • Перемістити — drag&drop по полотну
  • Змінити розмір — потягнути за кутовий маркер
  • Властивості — права панель при виділеному елементі
  • Видалити — кнопка Delete або кнопка в панелі властивостей
Зберігання: зміни зберігаються кнопкою Зберегти. Автозбереження відсутнє.

Параметри (точки)

Кожен параметр системи — це точка з унікальним числовим ID. Зберігається у Redis як хеш point:NNN.

Структура точки

  • id — числовий ідентифікатор
  • object — об'єкт (наприклад, home, boiler)
  • system — підсистема (heating, gas, power…)
  • pointname — назва параметра (pressure, temperature…)
  • value — поточне значення
  • unit — одиниці виміру (°C, bar, A…)
  • quality — якість сигналу
  • ts — Unix timestamp останнього оновлення

Список усіх точок із поточними значеннями доступний на сторінці Всі параметри.

Якість сигналу

Кожна точка має поле quality, яке впливає на колір елементів на екрані.

GOOD — норма
WARN — попередження
ALARM — аварія
UNCERT — невизначено
INIT — ініціалізація
NODATA — немає даних

Елементи Значення точки, Індикатор та Підсистема автоматично змінюють свій колір залежно від якості прив'язаного параметра.

Симулятор

Контейнер infrabox-simulator генерує тестові значення для всіх точок і публікує їх через MQTT. Дозволяє тестувати екрани без підключення реального обладнання.

Увімкнення / вимкнення

Керується з головного екрану: ☰ Меню → Налаштування → Симулятор. Перемикач призупиняє або відновлює контейнер (стан зберігається).

Увага: при увімкненому симуляторі реальні дані можуть перезаписуватись симульованими, якщо обидва джерела публікують одні й ті самі точки.

Архіватор

Контейнер infrabox-arch безперервно записує значення всіх точок у файлові томи. Доступний за адресою /arch/.

  • Фільтрація за об'єктом, підсистемою, пошук за назвою
  • Вибір кількох параметрів для порівняння трендів
  • Побудова графіків за довільний проміжок часу

Адміністрування

Сторінка /admin доступна після авторизації. Дозволяє управляти обліковими записами користувачів.

Для входу перейдіть на /login. Після авторизації в хедері з'явиться ім'я користувача, а кнопка Вийти завершить сесію.
